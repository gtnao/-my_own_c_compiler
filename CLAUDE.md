# プロジェクトルール

## プロジェクト概要

- chibicc（Rui Ueyama）にインスパイアされた、Rustで書くCコンパイラ
- ターゲット: x86-64 Linux, AT&T構文
- 外部依存なし（pure Rust）

## 実装計画

- **`PLAN.md` に全体の実装計画がある**。各ステップの内容はそこを参照すること

## 実装の進め方

- **1ステップずつ実装すること**：複数ステップの機能を一度に実装してはならない。1ステップの実装→テスト→docs→コミットを完了してから次のステップに進む
- **Phaseが終わるまで作業を止めないこと**
- **各ステップのワークフロー**：
  1. そのステップで必要な変更だけを実装する
  2. ビルド＆テストを通す
  3. `docs/step_X_Y.md` に解説ドキュメントを書く
  4. 実装とdocsをまとめてコミットする
  5. `git push` する
  6. 次のステップに進む

## ドキュメント

- **各ステップごとに `docs/step_X_Y.md` を必ず書くこと**
- docsの内容は淡白にせず、詳しく解説する（アセンブリの動作原理、レジスタの意味、なぜそうするのかの理由まで含める）
- docsはコミットに含めること（実装とdocsを分離しない）

## コミット

- コミットメッセージは英語
- 各ステップの実装＋docsをまとめて1コミットにする
- **コミットしたら必ず `git push` すること**

## ビルド・テスト

- ビルド: `cargo build`
- ユニットテスト: `cargo test`
- 統合テスト: `bash tests/test.sh`

## プロジェクト構成

- `src/main.rs` — CLIエントリポイント、コンパイルパイプラインの統合
- `src/token.rs` — トークン定義（TokenKind, Token）
- `src/lexer.rs` — 字句解析（トークナイザ）
- `src/ast.rs` — AST（抽象構文木）の定義
- `src/parser.rs` — 構文解析（再帰下降パーサー）
- `src/codegen.rs` — x86-64コード生成
- `src/error.rs` — エラー報告（ソース位置付き）
- `tests/test.sh` — 統合テストスクリプト
- `docs/` — ステップごとの解説ドキュメント
