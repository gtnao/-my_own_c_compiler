# ステップ 14.26: typedef名を使ったキャスト

## 概要

キャスト式でtypedef名をサポートするよう修正する。従来は `(TypedefName *)expr` がエラーになっていた。キャストの検出が型キーワードのみをチェックし、typedef名をチェックしていなかったためである。

## バグの内容

パーサーの `unary()` 関数は以下のチェックでキャスト式を検出していた:
```rust
if self.current().kind == TokenKind::LParen
    && self.pos + 1 < self.tokens.len()
    && Self::is_type_keyword(&self.tokens[self.pos + 1].kind)
```

`is_type_keyword()` は組み込みの型キーワード（`int`、`char`、`struct`など）のみをチェックし、typedef名はチェックしていなかった。そのため `(List *)malloc(...)` はキャストとして認識されず、関数呼び出しとしてパースされてエラーになっていた。

## 修正内容

キャスト検出をtypedef名もチェックする `is_type_start()` を使用するように変更した:

```rust
// Before:
&& Self::is_type_keyword(&self.tokens[self.pos + 1].kind)

// After:
&& self.is_type_start(&self.tokens[self.pos + 1].kind)
```

`is_type_start()` は以下の場合にtrueを返す:
- すべての型キーワード（`is_type_keyword()` 経由）
- typedef名（`self.typedefs.contains_key(name)` 経由）
- `va_list` / `__builtin_va_list`

## この修正が重要な理由

PostgreSQLではtypedef名へのキャストが頻繁に行われる:

```c
typedef struct MemoryContextData *MemoryContext;
MemoryContext ctx = (MemoryContext)palloc(sizeof(struct MemoryContextData));

typedef unsigned long Datum;
Datum d = (Datum)some_pointer;
```

## テストケース

```c
typedef int MyInt; int main() { long x = 10; return (MyInt)x; }     // → 10
typedef struct { int val; } S;
extern void *malloc(unsigned long);
int main() { S *p = (S *)malloc(sizeof(S)); p->val = 42; return p->val; }  // → 42
```
