# Step 14.26: Cast with Typedef Names

## Overview

Fix cast expressions to support typedef names. Previously, `(TypedefName *)expr` would fail because the cast detection only checked for type keywords, not typedef names.

## The Bug

The `unary()` function in the parser detects cast expressions by checking:
```rust
if self.current().kind == TokenKind::LParen
    && self.pos + 1 < self.tokens.len()
    && Self::is_type_keyword(&self.tokens[self.pos + 1].kind)
```

`is_type_keyword()` only checks for built-in type keywords (`int`, `char`, `struct`, etc.) but not typedef names. So `(List *)malloc(...)` would not be recognized as a cast and would be parsed as a function call, producing an error.

## Fix

Changed the cast detection to use `is_type_start()` which also checks typedef names:

```rust
// Before:
&& Self::is_type_keyword(&self.tokens[self.pos + 1].kind)

// After:
&& self.is_type_start(&self.tokens[self.pos + 1].kind)
```

`is_type_start()` returns true for:
- All type keywords (via `is_type_keyword()`)
- Typedef names (via `self.typedefs.contains_key(name)`)
- `va_list` / `__builtin_va_list`

## Why This Matters

PostgreSQL casts to typedef names constantly:

```c
typedef struct MemoryContextData *MemoryContext;
MemoryContext ctx = (MemoryContext)palloc(sizeof(struct MemoryContextData));

typedef unsigned long Datum;
Datum d = (Datum)some_pointer;
```

## Test Cases

```c
typedef int MyInt; int main() { long x = 10; return (MyInt)x; }     // → 10
typedef struct { int val; } S;
extern void *malloc(unsigned long);
int main() { S *p = (S *)malloc(sizeof(S)); p->val = 42; return p->val; }  // → 42
```
