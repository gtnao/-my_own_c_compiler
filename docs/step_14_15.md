# Step 14.15: GCC Statement Expressions `({ ... })`

## Overview

Add support for GCC statement expressions — a compound statement enclosed in parentheses that evaluates to the value of its last expression statement. This is `({ stmt1; stmt2; expr; })`.

## Why This Matters

PostgreSQL extensively uses statement expressions in macros for safe multi-statement operations:

```c
#define Max(a, b) \
    ({ typeof(a) _a = (a); typeof(b) _b = (b); _a > _b ? _a : _b; })
#define VARATT_IS_1B(ptr) \
    ({ struct varattrib_1b *_ptr = (ptr); _ptr->va_header & 0x01; })
```

Statement expressions allow declaring local variables inside an expression, avoiding issues with multiple evaluation of macro arguments.

## Implementation

### AST

Added `Expr::StmtExpr(Vec<Stmt>)` variant — holds a list of statements whose last expression statement provides the value.

### Parser

In `primary()`, when `(` is followed by `{`, it's parsed as a statement expression:

```rust
TokenKind::LParen => {
    self.advance();
    if self.current().kind == TokenKind::LBrace {
        self.advance();
        self.enter_scope();
        let mut stmts = Vec::new();
        while self.current().kind != TokenKind::RBrace {
            stmts.push(self.stmt());
        }
        self.expect(TokenKind::RBrace);
        self.leave_scope();
        self.expect(TokenKind::RParen);
        Expr::StmtExpr(stmts)
    } else {
        // regular parenthesized expression
        let node = self.expr();
        self.expect(TokenKind::RParen);
        node
    }
}
```

Key details:
- A new scope is entered for variable declarations inside `({ ... })`
- All statements are parsed until `}`
- Variables declared inside are scoped to the statement expression

### Code Generation

All statements are generated sequentially. The last expression statement's value naturally remains in `%rax`:

```rust
Expr::StmtExpr(stmts) => {
    for s in stmts {
        self.gen_stmt(s);
    }
}
```

### Type Inference

The type of a statement expression is the type of its last expression statement:

```rust
Expr::StmtExpr(stmts) => {
    if let Some(Stmt::ExprStmt(expr)) = stmts.last() {
        return self.expr_type(expr);
    }
    Type::int_type()
}
```

## Behavior

- `({ 5; })` evaluates to 5
- `({ 1; 2; 3; })` evaluates to 3 (last expression)
- `({ int x = 5; x + 5; })` evaluates to 10
- Variables declared inside are block-scoped (not visible outside)

## Test Cases

```c
int main() { return ({ 5; }); }                                           // → 5
int main() { return ({ 1; 2; 3; }); }                                     // → 3
int main() { int a = ({ int x = 5; x + 5; }); return a; }                 // → 10
int main() { int a = 3; int b = ({ int c = 4; a + c; }); return b; }      // → 7
```
