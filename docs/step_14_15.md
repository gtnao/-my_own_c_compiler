# ステップ 14.15: GCC 文式（Statement Expressions）`({ ... })`

## 概要

GCC 文式のサポートを追加する。文式とは、括弧で囲まれた複合文であり、最後の式文の値に評価される。構文は `({ stmt1; stmt2; expr; })` である。

## なぜ必要か

PostgreSQL では安全な複数文操作のためにマクロの中で文式を広く使用している:

```c
#define Max(a, b) \
    ({ typeof(a) _a = (a); typeof(b) _b = (b); _a > _b ? _a : _b; })
#define VARATT_IS_1B(ptr) \
    ({ struct varattrib_1b *_ptr = (ptr); _ptr->va_header & 0x01; })
```

文式を使えば式の内部でローカル変数を宣言でき、マクロ引数の多重評価の問題を回避できる。

## 実装

### AST

`Expr::StmtExpr(Vec<Stmt>)` バリアントを追加した。文のリストを保持し、最後の式文がその値を提供する。

### 構文解析

`primary()` において、`(` の後に `{` が続く場合、文式として解析される:

```rust
TokenKind::LParen => {
    self.advance();
    if self.current().kind == TokenKind::LBrace {
        self.advance();
        self.enter_scope();
        let mut stmts = Vec::new();
        while self.current().kind != TokenKind::RBrace {
            stmts.push(self.stmt());
        }
        self.expect(TokenKind::RBrace);
        self.leave_scope();
        self.expect(TokenKind::RParen);
        Expr::StmtExpr(stmts)
    } else {
        // regular parenthesized expression
        let node = self.expr();
        self.expect(TokenKind::RParen);
        node
    }
}
```

重要なポイント:
- `({ ... })` 内部の変数宣言のために新しいスコープに入る
- `}` までのすべての文が解析される
- 内部で宣言された変数はその文式のスコープに限定される

### コード生成

すべての文が順番に生成される。最後の式文の値は自然に `%rax` に残る:

```rust
Expr::StmtExpr(stmts) => {
    for s in stmts {
        self.gen_stmt(s);
    }
}
```

### 型推論

文式の型は、最後の式文の型である:

```rust
Expr::StmtExpr(stmts) => {
    if let Some(Stmt::ExprStmt(expr)) = stmts.last() {
        return self.expr_type(expr);
    }
    Type::int_type()
}
```

## 動作

- `({ 5; })` は 5 に評価される
- `({ 1; 2; 3; })` は 3 に評価される（最後の式）
- `({ int x = 5; x + 5; })` は 10 に評価される
- 内部で宣言された変数はブロックスコープである（外部からは見えない）

## テストケース

```c
int main() { return ({ 5; }); }                                           // → 5
int main() { return ({ 1; 2; 3; }); }                                     // → 3
int main() { int a = ({ int x = 5; x + 5; }); return a; }                 // → 10
int main() { int a = 3; int b = ({ int c = 4; a + c; }); return b; }      // → 7
```
