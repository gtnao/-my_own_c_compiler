# Step 12.11: _Alignof and _Alignas

## Overview

Implement C11 alignment operators:
- `_Alignof(type)` — returns the alignment requirement of a type (compile-time constant)
- `_Alignas(N)` — specifies alignment for a variable declaration (parsed but alignment not enforced)

## _Alignof

`_Alignof(type)` evaluates to a compile-time constant representing the alignment in bytes of the given type. It is folded directly to `Expr::Num(align)` during parsing.

### Implementation

Added `Alignof` token kind and lexer recognition for `_Alignof`. In the parser's `unary()` method:

```rust
TokenKind::Alignof => {
    self.advance();
    self.expect(TokenKind::LParen);
    let ty = self.parse_type();
    self.expect(TokenKind::RParen);
    return Expr::Num(ty.align() as i64);
}
```

### Values

| Type | `_Alignof` |
|------|-----------|
| `char` | 1 |
| `short` | 2 |
| `int` | 4 |
| `long` | 8 |
| `int *` (any pointer) | 8 |
| `struct { int x; long y; }` | 8 (max member align) |

## _Alignas

`_Alignas(N)` or `_Alignas(type)` specifies the minimum alignment for a variable. It is parsed and consumed but the alignment enforcement is currently simplified (the specified alignment is acknowledged but not applied to stack layout).

### Parsing

`_Alignas` is treated as a type qualifier. When encountered in `parse_type()`, the parser consumes `_Alignas(...)` including the parenthesized argument:

```rust
if self.current().kind == TokenKind::Alignas {
    self.advance(); // _Alignas
    self.expect(TokenKind::LParen);
    // Skip alignment value (number or type)
    if self.is_type_start(...) {
        self.parse_type();
    } else {
        self.advance(); // skip number
    }
    self.expect(TokenKind::RParen);
}
```

This allows code using `_Alignas` to compile without errors, even though the actual alignment may not be enforced beyond the type's natural alignment.

## Test Cases

```c
return _Alignof(int);      // => 4
return _Alignof(long);     // => 8
return _Alignof(char);     // => 1
return _Alignof(int *);    // => 8
_Alignas(16) int x = 5; return x;  // => 5 (compiles, alignment not enforced)
```
