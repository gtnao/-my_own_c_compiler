# ステップ 12.11: _Alignof と _Alignas

## 概要

C11 のアライメント演算子を実装する:
- `_Alignof(type)` — 型のアライメント要件を返す（コンパイル時定数）
- `_Alignas(N)` — 変数宣言のアライメントを指定する（パースのみで、アライメントは強制されない）

## _Alignof

`_Alignof(type)` は、指定された型のアライメント（バイト単位）を表すコンパイル時定数に評価される。パース時に直接 `Expr::Num(align)` に畳み込まれる。

### 実装

`Alignof` トークン種別を追加し、レキサーで `_Alignof` を認識するようにした。パーサーの `unary()` メソッドでの処理:

```rust
TokenKind::Alignof => {
    self.advance();
    self.expect(TokenKind::LParen);
    let ty = self.parse_type();
    self.expect(TokenKind::RParen);
    return Expr::Num(ty.align() as i64);
}
```

### 各型の値

| 型 | `_Alignof` |
|------|-----------|
| `char` | 1 |
| `short` | 2 |
| `int` | 4 |
| `long` | 8 |
| `int *`（任意のポインタ） | 8 |
| `struct { int x; long y; }` | 8（メンバの最大アライメント） |

## _Alignas

`_Alignas(N)` または `_Alignas(type)` は変数の最小アライメントを指定する。パースされ消費されるが、アライメントの強制は現在簡略化されている（指定されたアライメントは認識されるが、スタックレイアウトには適用されない）。

### パース

`_Alignas` は型修飾子として扱われる。`parse_type()` で検出されると、パーサーは括弧内の引数を含む `_Alignas(...)` を消費する:

```rust
if self.current().kind == TokenKind::Alignas {
    self.advance(); // _Alignas
    self.expect(TokenKind::LParen);
    // Skip alignment value (number or type)
    if self.is_type_start(...) {
        self.parse_type();
    } else {
        self.advance(); // skip number
    }
    self.expect(TokenKind::RParen);
}
```

これにより、`_Alignas` を使用するコードがエラーなくコンパイルできる。ただし、実際のアライメントは型の自然なアライメントを超えて強制されない場合がある。

## テストケース

```c
return _Alignof(int);      // => 4
return _Alignof(long);     // => 8
return _Alignof(char);     // => 1
return _Alignof(int *);    // => 8
_Alignas(16) int x = 5; return x;  // => 5 (compiles, alignment not enforced)
```
