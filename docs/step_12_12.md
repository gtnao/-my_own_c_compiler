# Step 12.12: _Generic Selection Expression

## Overview

Implement C11's `_Generic` keyword, which selects an expression based on the type of a controlling expression at compile time:

```c
_Generic(x, int: 1, long: 2, default: 0)
```

If `x` is `int`, the result is `1`. If `x` is `long`, the result is `2`. Otherwise, the `default` case is selected.

## Syntax

```
_Generic(controlling-expression, association-list)
association-list = type-name : assignment-expression
                 | default : assignment-expression
                 | association-list , type-name : assignment-expression
```

## Implementation

### Parsing

`_Generic` is parsed in `primary()` and resolved at parse time:

1. Parse the controlling expression
2. Infer its type using `infer_type()`
3. Iterate through the type associations
4. Return the expression matching the controlling type, or the `default` expression

```rust
fn parse_generic(&mut self) -> Expr {
    self.expect(TokenKind::LParen);
    let ctrl_expr = self.assign();
    let ctrl_ty = self.infer_type(&ctrl_expr);
    self.expect(TokenKind::Comma);

    let mut result = None;
    let mut default_expr = None;

    loop {
        if self.current().kind == TokenKind::Default {
            self.advance();
            self.expect(TokenKind::Colon);
            default_expr = Some(self.assign());
        } else {
            let assoc_ty = self.parse_type();
            self.expect(TokenKind::Colon);
            let expr = self.assign();
            if result.is_none() && self.types_match(&ctrl_ty, &assoc_ty) {
                result = Some(expr);
            }
        }
        if self.current().kind == TokenKind::Comma {
            self.advance();
        } else {
            break;
        }
    }
    self.expect(TokenKind::RParen);
    result.or(default_expr).unwrap_or(Expr::Num(0))
}
```

### Type Inference (`infer_type`)

A best-effort type inference method for expressions:

| Expression | Inferred Type |
|---|---|
| `Num(_)` | `int` |
| `StrLit(_)` | `char *` |
| `Var(name)` | looked up from scopes/globals |
| `Deref(ptr)` | base type of pointer |
| `Addr(expr)` | pointer to expr's type |
| `Cast { ty, .. }` | the cast's target type |
| other | `int` (fallback) |

### Type Matching (`types_match`)

Two types match if they have the same `TypeKind` variant and signedness. Pointers match if their base types match. Arrays and pointers are interchangeable for matching purposes (array-to-pointer decay).

## Use Cases

`_Generic` is primarily used in C11 to implement type-generic macros like `<tgmath.h>`:

```c
#define abs(x) _Generic((x), \
    int: abs_int, \
    long: abs_long, \
    double: abs_double)(x)
```

## Test Cases

```c
int x = 0;  return _Generic(x, int: 1, long: 2, default: 0);   // => 1

char x = 0; return _Generic(x, int: 1, long: 2, default: 0);   // => 0 (default)
```
