# Step 14.22: Built-in Fixed-Width Integer Types

## Overview

Register standard C fixed-width integer types (`int8_t`, `int16_t`, `int32_t`, `int64_t`, etc.) as built-in typedefs so they can be used without `#include <stdint.h>`.

## Why This Matters

PostgreSQL uses fixed-width integer types extensively throughout its codebase:

```c
typedef int8_t int8;
typedef int16_t int16;
typedef int32_t int32;
typedef uint32_t uint32;
typedef int64_t int64;
typedef size_t Size;
typedef uintptr_t Datum;
```

These types come from `<stdint.h>` in standard C, but since our compiler doesn't have full standard library header support, we register them as built-in typedefs.

## Implementation

In the `Parser::new()` constructor, pre-populate the `typedefs` HashMap with standard fixed-width type mappings:

```rust
let mut typedefs = HashMap::new();
typedefs.insert("int8_t".to_string(), Type::char_type());      // 1 byte signed
typedefs.insert("int16_t".to_string(), Type::short_type());     // 2 bytes signed
typedefs.insert("int32_t".to_string(), Type::int_type());       // 4 bytes signed
typedefs.insert("int64_t".to_string(), Type::long_type());      // 8 bytes signed
typedefs.insert("uint8_t".to_string(), Type::uchar());          // 1 byte unsigned
typedefs.insert("uint16_t".to_string(), Type::ushort());        // 2 bytes unsigned
typedefs.insert("uint32_t".to_string(), Type::uint());          // 4 bytes unsigned
typedefs.insert("uint64_t".to_string(), Type::ulong());         // 8 bytes unsigned
typedefs.insert("size_t".to_string(), Type::ulong());           // pointer-sized unsigned
typedefs.insert("ssize_t".to_string(), Type::long_type());      // pointer-sized signed
typedefs.insert("intptr_t".to_string(), Type::long_type());     // pointer-sized signed
typedefs.insert("uintptr_t".to_string(), Type::ulong());        // pointer-sized unsigned
typedefs.insert("ptrdiff_t".to_string(), Type::long_type());    // pointer difference
```

### Type Sizes on x86-64 Linux

| Type | Size | Mapping |
|------|------|---------|
| `int8_t` / `uint8_t` | 1 byte | `char` / `unsigned char` |
| `int16_t` / `uint16_t` | 2 bytes | `short` / `unsigned short` |
| `int32_t` / `uint32_t` | 4 bytes | `int` / `unsigned int` |
| `int64_t` / `uint64_t` | 8 bytes | `long` / `unsigned long` |
| `size_t` / `uintptr_t` | 8 bytes | `unsigned long` (LP64) |
| `ssize_t` / `intptr_t` / `ptrdiff_t` | 8 bytes | `long` (LP64) |

These are correct for the x86-64 LP64 data model used on Linux.

## Test Cases

```c
int main() { int8_t a = 1; return a; }           // → 1
int main() { int16_t a = 2; return a; }          // → 2
int main() { return sizeof(int32_t); }            // → 4
int main() { return sizeof(int64_t); }            // → 8
int main() { return sizeof(uint64_t); }           // → 8
int main() { return sizeof(size_t); }             // → 8
int main() { return sizeof(ptrdiff_t); }          // → 8
```
