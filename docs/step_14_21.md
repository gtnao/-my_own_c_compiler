# Step 14.21: Typedef with Forward-Declared Struct Resolution

## Overview

Fix the issue where a typedef referencing a forward-declared (empty) struct would not be updated when the struct is later fully defined.

## Why This Matters

PostgreSQL uses the pattern `typedef struct Foo *FooPtr;` before the full struct definition extensively:

```c
struct MemoryContextData;
typedef struct MemoryContextData *MemoryContext;

struct MemoryContextData {
    int type;
    MemoryContext parent;
    MemoryContext firstchild;
    char *name;
};
```

Without this fix, `MemoryContext` would point to an empty struct with no members, causing panics when accessing members via `->`.

## The Problem

The forward declaration flow:

1. `struct Foo;` — creates empty struct placeholder `Struct(Vec::new())` in `struct_tags`
2. `typedef struct Foo *FooPtr;` — stores `Ptr(Struct(Vec::new()))` in `typedefs`
3. `struct Foo { int x; int y; };` — updates `struct_tags["Foo"]` with full definition

At step 3, the typedef still holds a pointer to the empty struct. When code later uses `FooPtr p = &f; p->x`, the codegen tries to find member `x` in an empty member list and panics.

## Implementation

### `update_typedefs_with_struct()`

After a struct tag is registered with its full definition, scan all typedefs and replace any empty struct placeholders with the full definition:

```rust
fn update_typedefs_with_struct(&mut self, full_ty: &Type) {
    let keys: Vec<String> = self.typedefs.keys().cloned().collect();
    for key in keys {
        let ty = self.typedefs.get(&key).unwrap().clone();
        if let Some(updated) = Self::replace_empty_struct(&ty, full_ty) {
            self.typedefs.insert(key, updated);
        }
    }
}
```

### `replace_empty_struct()`

Recursive helper that walks a type tree and replaces empty structs with the full definition:

```rust
fn replace_empty_struct(ty: &Type, full_ty: &Type) -> Option<Type> {
    match &ty.kind {
        TypeKind::Struct(members) if members.is_empty() => {
            if let TypeKind::Struct(full_members) = &full_ty.kind {
                if !full_members.is_empty() {
                    return Some(full_ty.clone());
                }
            }
            None
        }
        TypeKind::Ptr(base) => {
            Self::replace_empty_struct(base, full_ty)
                .map(|updated| Type { kind: TypeKind::Ptr(Box::new(updated)), .. })
        }
        TypeKind::Array(base, size) => {
            Self::replace_empty_struct(base, full_ty)
                .map(|updated| Type { kind: TypeKind::Array(Box::new(updated), *size), .. })
        }
        _ => None,
    }
}
```

This handles nested types like `Ptr(Ptr(Struct(empty)))` or `Array(Ptr(Struct(empty)), N)`.

### Integration Point

Called right after registering a struct tag with its full definition:

```rust
if let Some(ref tag) = tag_name {
    self.struct_tags.insert(tag.clone(), ty.clone());
    self.update_typedefs_with_struct(&ty);
}
```

## Test Cases

```c
struct Foo; typedef struct Foo *FooPtr; struct Foo { int x; int y; };
int main() { struct Foo f; f.x=3; f.y=7; FooPtr p=&f; return p->x + p->y; }  // → 10

struct Ctx; typedef struct Ctx *CtxPtr;
struct Ctx { int type; CtxPtr parent; };
int main() { struct Ctx c; c.type=5; c.parent=0; CtxPtr p=&c; return p->type; }  // → 5
```
