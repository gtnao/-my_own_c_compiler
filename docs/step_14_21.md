# ステップ 14.21: 前方宣言された構造体を参照するtypedefの解決

## 概要

typedefが前方宣言された（空の）構造体を参照している場合、構造体が後から完全に定義されてもtypedefが更新されない問題を修正する。

## この修正が重要な理由

PostgreSQLでは、完全な構造体定義よりも前に `typedef struct Foo *FooPtr;` というパターンが多用されている:

```c
struct MemoryContextData;
typedef struct MemoryContextData *MemoryContext;

struct MemoryContextData {
    int type;
    MemoryContext parent;
    MemoryContext firstchild;
    char *name;
};
```

この修正がないと、`MemoryContext`はメンバを持たない空の構造体を指したままになり、`->`でメンバにアクセスする際にパニックが発生する。

## 問題の詳細

前方宣言の処理フロー:

1. `struct Foo;` — `struct_tags`に空の構造体プレースホルダ `Struct(Vec::new())` を作成
2. `typedef struct Foo *FooPtr;` — `typedefs`に `Ptr(Struct(Vec::new()))` を格納
3. `struct Foo { int x; int y; };` — `struct_tags["Foo"]`を完全な定義で更新

ステップ3の時点で、typedefはまだ空の構造体へのポインタを保持している。後でコードが `FooPtr p = &f; p->x` を使うと、コード生成が空のメンバリストからメンバ `x` を探そうとしてパニックする。

## 実装

### `update_typedefs_with_struct()`

構造体タグの完全な定義が登録された後、すべてのtypedefをスキャンし、空の構造体プレースホルダを完全な定義に置き換える:

```rust
fn update_typedefs_with_struct(&mut self, full_ty: &Type) {
    let keys: Vec<String> = self.typedefs.keys().cloned().collect();
    for key in keys {
        let ty = self.typedefs.get(&key).unwrap().clone();
        if let Some(updated) = Self::replace_empty_struct(&ty, full_ty) {
            self.typedefs.insert(key, updated);
        }
    }
}
```

### `replace_empty_struct()`

型ツリーを再帰的に走査し、空の構造体を完全な定義に置き換えるヘルパー関数:

```rust
fn replace_empty_struct(ty: &Type, full_ty: &Type) -> Option<Type> {
    match &ty.kind {
        TypeKind::Struct(members) if members.is_empty() => {
            if let TypeKind::Struct(full_members) = &full_ty.kind {
                if !full_members.is_empty() {
                    return Some(full_ty.clone());
                }
            }
            None
        }
        TypeKind::Ptr(base) => {
            Self::replace_empty_struct(base, full_ty)
                .map(|updated| Type { kind: TypeKind::Ptr(Box::new(updated)), .. })
        }
        TypeKind::Array(base, size) => {
            Self::replace_empty_struct(base, full_ty)
                .map(|updated| Type { kind: TypeKind::Array(Box::new(updated), *size), .. })
        }
        _ => None,
    }
}
```

これにより、`Ptr(Ptr(Struct(empty)))` や `Array(Ptr(Struct(empty)), N)` のようなネストされた型も処理できる。

### 統合箇所

構造体タグの完全な定義を登録した直後に呼び出される:

```rust
if let Some(ref tag) = tag_name {
    self.struct_tags.insert(tag.clone(), ty.clone());
    self.update_typedefs_with_struct(&ty);
}
```

## テストケース

```c
struct Foo; typedef struct Foo *FooPtr; struct Foo { int x; int y; };
int main() { struct Foo f; f.x=3; f.y=7; FooPtr p=&f; return p->x + p->y; }  // → 10

struct Ctx; typedef struct Ctx *CtxPtr;
struct Ctx { int type; CtxPtr parent; };
int main() { struct Ctx c; c.type=5; c.parent=0; CtxPtr p=&c; return p->type; }  // → 5
```
