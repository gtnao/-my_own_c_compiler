# Step 14.25: Fix Typedef Forward-Declared Struct Resolution (Tag-Specific)

## Overview

Fix a bug where `update_typedefs_with_struct()` would incorrectly replace ALL empty structs in typedefs with the newly defined struct, even when the typedefs referenced different struct tags.

## The Bug

When multiple structs are forward-declared, and then defined one by one, the first struct definition would corrupt typedefs that reference other still-forward-declared structs:

```c
struct A;                          // forward-declare A (empty struct)
struct B;                          // forward-declare B (empty struct)
typedef struct A *APtr;            // typedefs["APtr"] = Ptr(empty struct)
typedef struct B *BPtr;            // typedefs["BPtr"] = Ptr(empty struct)

struct A { int x; };               // defines A
// BUG: update_typedefs_with_struct replaces ALL empty structs with A's definition
// Now BPtr = Ptr(struct A) instead of Ptr(empty struct B)!

struct B { int y; int z; };        // defines B, but BPtr was already corrupted
```

This caused member access through the wrong typedef to produce incorrect offsets or panics.

## Fix

### 1. Track typedef-to-struct-tag mapping

Added `typedef_struct_tags: HashMap<String, String>` to the Parser, mapping typedef names to their associated struct tag names. Also added `last_struct_tag: Option<String>` to track the most recently parsed struct/union tag.

### 2. Record struct tag at typedef registration

At every typedef registration point (both top-level and statement-level), if a struct tag was involved, record the mapping:

```rust
if let Some(ref tag) = self.last_struct_tag {
    self.typedef_struct_tags.insert(name.clone(), tag.clone());
}
self.typedefs.insert(name, ty);
```

### 3. Tag-specific typedef updating

Changed `update_typedefs_with_struct()` to `update_typedefs_for_tag()`, which only updates typedefs that reference the specific struct tag being defined:

```rust
fn update_typedefs_for_tag(&mut self, tag: &str, full_ty: &Type) {
    let typedef_names: Vec<String> = self.typedef_struct_tags.iter()
        .filter(|(_, v)| *v == tag)
        .map(|(k, _)| k.clone())
        .collect();
    for name in typedef_names {
        let ty = self.typedefs.get(&name).unwrap().clone();
        if let Some(updated) = Self::replace_empty_struct(&ty, full_ty) {
            self.typedefs.insert(name, updated);
        }
    }
}
```

Now when `struct A` is defined, only typedefs that reference tag "A" are updated, leaving typedefs that reference tag "B" untouched.

## Test Cases

```c
// Multiple forward declarations with different struct tags
struct A; struct B;
typedef struct A *APtr; typedef struct B *BPtr;
struct A { int x; }; struct B { int y; int z; };
int main() { struct A a; a.x = 10; APtr p = &a; return p->x; }  // → 10
```
