# ステップ 12.10: フレキシブル配列メンバ

## 概要

構造体メンバに関連する2つの機能をサポートする:

1. **固定サイズ配列メンバ**: `struct { int x; int arr[3]; };`
2. **フレキシブル配列メンバ（FAM）**: `struct { int len; int data[]; };`

## 固定サイズ配列メンバ

構造体メンバとして明示的なサイズを持つ配列を使えるようになった:

```c
struct { int x; int arr[3]; } s;
s.arr[0] = 1; s.arr[1] = 2; s.arr[2] = 3;
return s.arr[1] + s.arr[0];  // => 3
```

メンバの型は `Array(Int, 3)` でサイズは12バイトとなる。構造体のオフセット計算は配列の全サイズとアライメントを考慮する。

## フレキシブル配列メンバ（C99）

フレキシブル配列メンバはサイズが指定されず、構造体の最後のメンバでなければならない:

```c
struct Header {
    int len;
    int data[];   // flexible array member — size 0
};
```

`sizeof(struct Header)` は固定メンバのみをカウントする（ここでは `len` の4バイト）。フレキシブル配列は構造体レイアウト上のサイズはゼロだが、構造体が余分な領域とともに確保された場合、ポインタ演算経由でアクセスできる:

```c
struct Header *p;
char buf[20];
p = (struct Header *)buf;  // overlay struct on buffer
p->len = 3;
p->data[0] = 42;           // access past the struct's nominal size
```

## パース

メンバ名をパースした後、パーサーは `[` があるかチェックする:

```rust
let mem_ty = if self.current().kind == TokenKind::LBracket {
    self.advance();
    if self.current().kind == TokenKind::RBracket {
        // Flexible array: name[]
        self.advance();
        Type::array_of(mem_ty, 0)   // size 0
    } else {
        // Fixed array: name[N]
        let len = /* parse number */;
        self.expect(TokenKind::RBracket);
        Type::array_of(mem_ty, len)
    }
} else {
    mem_ty
};
```

`Array(base, 0)` 型は `size() = base.size() * 0 = 0` となるため、構造体の合計サイズには影響しない。ただし、オフセットは正しく設定されるので、`data[i]` は `base_addr + offset + i * element_size` を計算し、十分なメモリが確保されていれば任意のインデックスで正しく動作する。

## テストケース

```c
// Fixed-size array member
struct { int x; int arr[3]; } s;
s.arr[0]=1; s.arr[1]=2; s.arr[2]=3;
return s.arr[1]+s.arr[0];  // => 3

// Flexible array member via pointer cast
struct { int len; int data[]; } *p;
char buf[20]; p = (void *)buf;
p->len = 3; p->data[0] = 4;
return p->data[0];  // => 4
```
