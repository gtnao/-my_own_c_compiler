# Step 1.6: 比較演算子

## 概要

`==`, `!=`, `<`, `<=`, `>`, `>=` の6つの比較演算子を追加する。パーサーに `equality` と `relational` の2つの優先順位レベルを追加し、コード生成では `cmp` + `setXX` + `movzb` パターンを使う。

## 文法規則の変更

```
expr       = equality
equality   = relational ("==" relational | "!=" relational)*    ← NEW
relational = add ("<" add | "<=" add | ">" add | ">=" add)*     ← NEW
add        = mul ("+" mul | "-" mul)*
mul        = unary ("*" unary | "/" unary | "%" unary)*
unary      = ("+" | "-") unary | primary
primary    = num | "(" expr ")"
```

**優先順位** (低い → 高い): `==`/`!=` < `<`/`<=`/`>`/`>=` < `+`/`-` < `*`/`/`/`%`

## レキサーの変更 — 2文字トークン

`==`, `!=`, `<=`, `>=` は2文字で1つのトークン。`peek_next()` で次の文字を先読みして判定する:

```rust
if ch == '=' && self.peek_next() == Some('=') {
    self.pos += 2;
    tokens.push(Token { kind: TokenKind::EqEq, pos });
    continue;
}
```

`<` と `<=` の区別: `<` の次が `=` なら `Le`、そうでなければ `Lt`。

## フラグレジスタ (FLAGS/RFLAGS) とは

x86-64 CPUには汎用レジスタ (`%rax`, `%rbp` 等) の他に、**フラグレジスタ** (RFLAGS) という特殊なレジスタがある。これはプログラマが直接値を入れるものではなく、`add`, `sub`, `cmp` などの算術命令が実行されるたびに **CPUが自動で更新する**。

フラグレジスタの中には複数のフラグ (1ビットのON/OFF) が詰まっている。比較に使う主要なフラグは以下の4つ:

| フラグ | 名前 | 意味 |
|--------|------|------|
| **ZF** | Zero Flag | 計算結果が 0 なら 1 |
| **SF** | Sign Flag | 計算結果の最上位ビット (= 符号ビット)。結果が負なら 1 |
| **CF** | Carry Flag | 符号なし演算で桁溢れ (繰り上がり/繰り下がり) が起きたら 1 |
| **OF** | Overflow Flag | 符号付き演算でオーバーフローが起きたら 1 |

### 具体例: `cmp` でフラグがどう変わるか

`cmp %rdi, %rax` は内部で `%rax - %rdi` を計算し、結果は捨てるがフラグだけ更新する。

**例1: `5 == 5`** (`%rax=5, %rdi=5`)
```
5 - 5 = 0
  → ZF = 1 (結果が0)
  → SF = 0 (結果は非負)
  → OF = 0
```

**例2: `3 < 5`** (`%rax=3, %rdi=5`)
```
3 - 5 = -2
  → ZF = 0 (結果は0ではない)
  → SF = 1 (結果が負)
  → OF = 0 (オーバーフローなし)
```

**例3: `5 > 3`** (`%rax=5, %rdi=3`)
```
5 - 3 = 2
  → ZF = 0
  → SF = 0 (結果は正)
  → OF = 0
```

### setXX が見ているもの

`setXX` 命令はフラグの組み合わせを見て、条件が成り立てば `%al` に 1、成り立たなければ 0 をセットする。

- **`sete`** (equal): `ZF == 1` か？ → 引き算の結果が0 = 両者が等しい
- **`setne`** (not equal): `ZF == 0` か？ → 引き算の結果が非0 = 等しくない
- **`setl`** (less): `SF != OF` か？ → 通常は「結果が負 = 左辺が小さい」。OFが絡むのはオーバーフロー時の補正のため
- **`setle`** (less or equal): `ZF == 1 || SF != OF` か？ → 等しいか小さい
- **`setg`** (greater): `ZF == 0 && SF == OF` か？ → 等しくなく、かつ左辺が大きい
- **`setge`** (greater or equal): `SF == OF` か？ → 左辺が大きいか等しい

#### なぜ `<` は単に SF=1 ではなく SF!=OF なのか

普通のケースでは `3 - 5 = -2` で `SF=1` となり、`SF=1` だけで「小さい」と判定できる。しかし **オーバーフロー** が起きると符号ビットが反転する:

```
例: -5 - 最大正の数 → 結果は正に見える (オーバーフロー)
  SF = 0 だが、実際は左辺の方が小さい
  OF = 1 (オーバーフロー発生)
  SF != OF → 1 (正しく "小さい" と判定)
```

`SF != OF` という条件にすることで、オーバーフローの有無に関わらず正しく比較できる。

## コード生成 — cmp + setXX + movzb パターン

比較演算は3命令で実現する:

```asm
  cmp %rdi, %rax     # rax - rdi を計算、結果は捨ててフラグだけ更新
  sete %al            # ZF=1 なら al=1, そうでなければ al=0
  movzb %al, %rax     # al (8bit) を rax (64bit) にゼロ拡張
```

### %al とは — レジスタの部分アクセス

`%rax` は64ビットのレジスタだが、その一部を別名でアクセスできる:

```
%rax  = |63 -------- 32|31 -------- 16|15 --- 8|7 ---- 0|
                                        %ah      %al
                                       |     %ax       |
                         |          %eax               |
         |                  %rax                       |
```

`%al` は `%rax` の最下位8ビット。`setXX` は歴史的な理由で8ビットレジスタにしか書き込めない。

### なぜ movzb が必要か

`setXX` は `%al` (8ビット) にしか書き込まない。`%rax` の上位56ビットには古い値が残っている可能性がある:

```
setXX 前: %rax = 0x0000DEADBEEF1234
setXX 後: %rax = 0x0000DEADBEEF1201  ← al だけ 01 に変わった
                                  ^^
movzb 後: %rax = 0x0000000000000001  ← 上位ビットがゼロクリアされた
```

`movzb` (Move with Zero-extend Byte) で上位ビットをゼロクリアし、正しい 0 または 1 の64ビット値にする。

### setXX バリエーション一覧

| 命令 | フラグ条件 | 意味 | C演算子 |
|------|-----------|------|--------|
| `sete` | ZF=1 | 等しい | `==` |
| `setne` | ZF=0 | 等しくない | `!=` |
| `setl` | SF!=OF | より小さい (符号付き) | `<` |
| `setle` | ZF=1 or SF!=OF | 以下 (符号付き) | `<=` |
| `setg` | ZF=0 and SF=OF | より大きい (符号付き) | `>` |
| `setge` | SF=OF | 以上 (符号付き) | `>=` |

## テスト

| 入力 | 期待値 | 検証内容 |
|------|--------|---------|
| `1==1` | 1 | 等価 (真) |
| `1==2` | 0 | 等価 (偽) |
| `1!=2` | 1 | 非等価 (真) |
| `1<2` | 1 | 未満 (真) |
| `1<1` | 0 | 未満 (偽: 等しい) |
| `1<=1` | 1 | 以下 (等しい場合も真) |
| `2>1` | 1 | 超過 (真) |
| `2>=2` | 1 | 以上 (等しい場合も真) |
| `5+1==6` | 1 | 算術式と比較の組み合わせ |
| `3*2>=5` | 1 | 乗算と比較の組み合わせ |
