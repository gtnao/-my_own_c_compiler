# Step 14.27: Empty Statement (Bare Semicolon)

## Overview

Add support for empty statements (a bare `;`). This allows patterns like `for (...);` (empty body) and standalone `;` in code.

## Why This Matters

PostgreSQL and general C code uses empty statements in several patterns:

```c
// Empty for body — all work done in the increment
for (i = 0; i < n; i++);

// Empty while body
while (*p++);

// Standalone semicolons (sometimes from macro expansions)
;
```

## The Bug

The parser's `stmt()` function had no case for `TokenKind::Semicolon`, so encountering a bare `;` would fall through to expression parsing, which would fail with "expected a number, identifier or '('".

## Fix

Added a `Semicolon` case at the beginning of `stmt()`:

```rust
TokenKind::Semicolon => {
    self.advance();
    Stmt::Block(vec![])  // empty statement as empty block
}
```

## Test Cases

```c
int main() { int i; for (i = 0; i < 10; i++); return i; }    // → 10
int main() { ; return 0; }                                     // → 0
int main() { int i = 0; while (i < 5) { i++; } ; return i; } // → 5
```
