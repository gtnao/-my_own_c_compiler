# ステップ 14.14: `typeof` / `__typeof__` 演算子

## 概要

`typeof`、`__typeof`、`__typeof__` 型演算子のサポートを追加する。これらにより、式や別の型の型を宣言における型指定子として使用できるようになる。

## なぜ必要か

PostgreSQL や GCC ヘッダではマクロの中で `typeof` が広く使われている:

```c
#define Max(a, b) \
    ({ typeof(a) _a = (a); typeof(b) _b = (b); _a > _b ? _a : _b; })
typeof(x) temp = x;
```

## 実装

### トークン

`TokenKind` に `Typeof` バリアントを追加した。

### 字句解析

3つの綴りを認識する:
- `typeof` — GCC 拡張（および C23 標準）
- `__typeof` — GCC 拡張
- `__typeof__` — GCC 拡張（二重アンダースコア形式）

### 構文解析

`typeof` は `parse_type()` の型マッチングセクションで処理される:

```rust
TokenKind::Typeof => {
    self.advance();
    self.expect(TokenKind::LParen);
    let ty = if self.is_type_start(&self.current().kind.clone()) {
        // typeof(type) — e.g., typeof(int)
        self.parse_type()
    } else {
        // typeof(expr) — e.g., typeof(x)
        let expr = self.expr();
        self.infer_type(&expr)
    };
    self.expect(TokenKind::RParen);
    ty
}
```

2つの形式に対応している:
1. **`typeof(type)`** — 型をそのまま返す（例: `typeof(int)` → `int`）
2. **`typeof(expr)`** — `infer_type()` を使って式の型を推論する（例: `x` が `int` の場合、`typeof(x)` → `int`）

`is_type_keyword()` および `stmt()` の型開始パターンに `Typeof` を追加した。

## 型推論

`infer_type()` メソッドは式の型を判定する:
- `Num(_)` → `int`
- `FloatLit(_)` → `double`
- `StrLit(_)` → `char *`
- `Var(name)` → ローカル変数/グローバル変数から検索
- `Deref(inner)` → ポインタの基本型
- `Addr(inner)` → 内側の型へのポインタ
- `Cast { ty, .. }` → キャスト先の型
- その他 → `int`（フォールバック）

## テストケース

```c
int main() { typeof(int) a = 4; return a; }                 // → 4
int main() { int x = 5; typeof(x) y = x; return y; }        // → 5
int main() { return sizeof(typeof(long)); }                  // → 8
int main() { return sizeof(__typeof__(int)); }               // → 4
int main() { int a = 3; __typeof(a) b = a; return b; }      // → 3
```
