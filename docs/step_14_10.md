# ステップ 14.10: `__extension__` キーワード

## 概要

GCC の `__extension__` キーワードのサポートを追加する。宣言の前でも式の前でも、出現する場所を問わず読み取られた後に無視される。

## なぜ必要か

GCC は非標準拡張に関する警告を抑制するために `__extension__` を使用する。PostgreSQL やシステムヘッダではマクロ定義の中で使われている:

```c
__extension__ typedef unsigned long long __uint64_t;
__extension__ ({ int __val = expr; __val; })
```

## 実装

### トークン

`TokenKind` に `Extension` バリアントを追加した。

### 字句解析

`__extension__` をキーワードとして認識し、`TokenKind::Extension` にマッピングする。

### 構文解析

`__extension__` は2箇所で読み取られ、無視される:

1. **型宣言の前** — `parse_type()` において `inline`、`_Noreturn`、`register` と並んで処理される:
   ```rust
   while matches!(self.current().kind,
       TokenKind::Inline | TokenKind::Noreturn | TokenKind::Register | TokenKind::Extension) {
       self.advance();
   }
   ```

2. **式の前** — `unary()` の先頭で処理される:
   ```rust
   while self.current().kind == TokenKind::Extension {
       self.advance();
   }
   ```

`is_type_keyword()` および `stmt()` の型開始パターンにも追加した。

## 動作

- `__extension__` は言語拡張に関する警告を抑制するための GCC 固有のキーワードである
- 本コンパイラには抑制すべき警告がないため、単に読み取って無視する
- 宣言の前（`__extension__ int a = 5;`）と式の前（`return __extension__ 3;`）の両方で動作する

## テストケース

```c
int main() { __extension__ int a = 5; return a; }  // → 5
int main() { return __extension__ 3; }              // → 3
```
