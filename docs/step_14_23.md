# Step 14.23: Fix `expr_type()` for Cast Expressions

## Overview

Fix a critical bug where `expr_type()` in the code generator didn't handle `Expr::Cast` nodes, causing incorrect memory load sizes when dereferencing through a cast (e.g., `*(int *)ptr`).

## The Bug

When code like `*(int *)void_ptr` was compiled, the deref codegen needed to know the dereferenced type to select the correct load instruction:
- `int` → `movslq (%rax), %rax` (sign-extending 4-byte load)
- `long`/pointer → `mov (%rax), %rax` (8-byte load)

The `expr_type()` function traces through expression nodes to determine types. For `Deref(Cast{ty: Ptr(Int), expr})`:

1. `expr_type(Deref(inner))` calls `expr_type(inner)` to get the pointer type
2. `expr_type(Cast{ty: Ptr(Int), ...})` had no dedicated case
3. Fell through to `_ => Type::long_type()` (the default)
4. Back in Deref, `Long` is not `Ptr` or `Array`, so again fell through to `Type::long_type()`
5. `emit_load_indirect(long)` emitted `mov (%rax), %rax` — an 8-byte load for 4-byte data

## Symptoms

The 8-byte load reads 4 bytes of the target variable plus 4 bytes of adjacent stack data (typically another variable or the saved `%rbp`). This produces a 64-bit value with garbage in the upper 32 bits.

When the result was stored in an `int` variable first, `movl %eax, -offset(%rbp)` truncated to 32 bits, masking the bug. But when used directly in a comparison like `if (func_call() > 0)`, the full 64-bit garbage value was compared, producing wrong results.

Example:
```c
int int_cmp(void *a, void *b) {
    return *(int *)a - *(int *)b;  // 8-byte loads → garbage in upper bits
}

int main() {
    int x = 10, y = 5;
    if (int_cmp(&x, &y) > 0) return 1;  // Failed: compared garbage 64-bit value
    return 0;
}
```

## Fix

Added `Expr::Cast` case to `expr_type()`:

```rust
Expr::Cast { ty, .. } => ty.clone(),
```

Now `expr_type(Cast{ty: Ptr(Int), ...})` correctly returns `Ptr(Int)`, and `Deref` extracts the base type `Int`, selecting `movslq (%rax), %rax` (the correct 4-byte sign-extending load).

## Generated Assembly Before/After

Before:
```asm
  mov (%rax), %rax     # 8-byte load — reads 4 bytes of garbage
```

After:
```asm
  movslq (%rax), %rax  # 4-byte load with sign extension — correct
```

## Test Cases

```c
int main() { int a=10; int b=5; void *pa=&a; void *pb=&b; return *(int*)pa - *(int*)pb; }  // → 5
int int_cmp(void *a, void *b) { return *(int*)a - *(int*)b; }
int main() { int x=10; int y=5; if (int_cmp(&x, &y) > 0) return 1; return 0; }              // → 1
```
