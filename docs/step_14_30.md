# Step 14.30: Function Pointer Call Through Any Expression

## Overview

Support calling function pointers through arbitrary expressions, not just struct member access. This enables patterns like `ops[i](args)`, `(*fp)(args)`, and any other expression that evaluates to a function pointer.

## The Bug

The `postfix()` loop handled function pointer calls only after `.` and `->` member access:

```rust
TokenKind::Dot => {
    // ... create Member node ...
    if self.current().kind == TokenKind::LParen {
        // Parse function pointer call through member
    }
}
TokenKind::Arrow => {
    // ... create Member node ...
    if self.current().kind == TokenKind::LParen {
        // Parse function pointer call through member
    }
}
```

After array subscript (`a[i]`), dereference (`*p`), or other expressions, a following `(` was not recognized as a function call. So `ops[0](10, 5)` would parse `ops[0]` as a subscript expression, then the `(10, 5)` would fall through to the next statement, causing errors.

## Fix

Added a generic `LParen` case in the `postfix()` loop:

```rust
TokenKind::LParen => {
    self.advance();
    let mut args = Vec::new();
    if self.current().kind != TokenKind::RParen {
        args.push(self.assign());
        while self.current().kind == TokenKind::Comma {
            self.advance();
            args.push(self.assign());
        }
    }
    self.expect(TokenKind::RParen);
    node = Expr::FuncPtrCall {
        fptr: Box::new(node),
        args,
    };
}
```

This handles all expressions that produce a function pointer value:
- `ops[i](args)` — array of function pointers
- `(*fp)(args)` — explicit dereference
- `get_handler()(args)` — function returning function pointer
- `table[i].handler(args)` — chained member + call (already worked via member case, but now doubly covered)

## Why This Matters

PostgreSQL uses function pointer tables extensively:

```c
typedef void (*NodeFunc)(Node *);
static NodeFunc dispatch_table[] = {
    handle_select,
    handle_insert,
    handle_update,
};

dispatch_table[node->type](node);
```

## Test Cases

```c
int add(int a, int b) { return a+b; }
typedef int (*BinOp)(int, int);
BinOp ops[2]; ops[0] = add;
return ops[0](10, 5);  // → 15

// Array of 3 function pointers
return ops[0](10,5) + ops[1](10,5) + ops[2](10,5);  // → 70 (15+5+50)
```
