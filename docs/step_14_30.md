# ステップ 14.30: 任意の式を介した関数ポインタ呼び出し

## 概要

構造体メンバアクセスだけでなく、任意の式を通じた関数ポインタの呼び出しをサポートする。これにより `ops[i](args)`、`(*fp)(args)` など、関数ポインタに評価される任意の式からの呼び出しが可能になる。

## バグの内容

`postfix()` ループでは、`.` と `->` のメンバアクセスの後にのみ関数ポインタ呼び出しを処理していた:

```rust
TokenKind::Dot => {
    // ... create Member node ...
    if self.current().kind == TokenKind::LParen {
        // Parse function pointer call through member
    }
}
TokenKind::Arrow => {
    // ... create Member node ...
    if self.current().kind == TokenKind::LParen {
        // Parse function pointer call through member
    }
}
```

配列添字（`a[i]`）、デリファレンス（`*p`）、その他の式の後に続く `(` は関数呼び出しとして認識されなかった。そのため `ops[0](10, 5)` では `ops[0]` が添字式としてパースされた後、`(10, 5)` が次の文にフォールスルーしてエラーが発生していた。

## 修正内容

`postfix()` ループに汎用的な `LParen` ケースを追加した:

```rust
TokenKind::LParen => {
    self.advance();
    let mut args = Vec::new();
    if self.current().kind != TokenKind::RParen {
        args.push(self.assign());
        while self.current().kind == TokenKind::Comma {
            self.advance();
            args.push(self.assign());
        }
    }
    self.expect(TokenKind::RParen);
    node = Expr::FuncPtrCall {
        fptr: Box::new(node),
        args,
    };
}
```

これにより、関数ポインタ値を生成するすべての式を処理できる:
- `ops[i](args)` — 関数ポインタの配列
- `(*fp)(args)` — 明示的なデリファレンス
- `get_handler()(args)` — 関数ポインタを返す関数
- `table[i].handler(args)` — メンバ + 呼び出しの連鎖（メンバケースで既に動作していたが、これで二重にカバーされる）

## この実装が重要な理由

PostgreSQLでは関数ポインタテーブルが多用されている:

```c
typedef void (*NodeFunc)(Node *);
static NodeFunc dispatch_table[] = {
    handle_select,
    handle_insert,
    handle_update,
};

dispatch_table[node->type](node);
```

## テストケース

```c
int add(int a, int b) { return a+b; }
typedef int (*BinOp)(int, int);
BinOp ops[2]; ops[0] = add;
return ops[0](10, 5);  // → 15

// Array of 3 function pointers
return ops[0](10,5) + ops[1](10,5) + ops[2](10,5);  // → 70 (15+5+50)
```
