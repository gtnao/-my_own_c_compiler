# Step 14.11: `__builtin_*` GCC Built-in Functions

## Overview

Add support for commonly used GCC built-in functions. These are handled as special cases in the parser and evaluated at compile time or simplified to their essential behavior.

## Why This Matters

PostgreSQL and system headers extensively use GCC built-in functions:

```c
#define likely(x)   __builtin_expect(!!(x), 1)
#define unlikely(x) __builtin_expect(!!(x), 0)
#define offsetof(type, member) __builtin_offsetof(type, member)
typedef __builtin_va_list va_list;
```

## Implemented Built-ins

### `__builtin_expect(expr, expected)`

Branch prediction hint. Returns `expr` unchanged â€” the `expected` argument is ignored since we don't perform branch prediction optimization.

```c
__builtin_expect(x > 0, 1)  // â†’ evaluates to (x > 0)
```

### `__builtin_constant_p(expr)`

Returns 1 if `expr` is a compile-time constant, 0 otherwise. Our compiler always returns 0 since we don't track constant expressions for this purpose.

```c
__builtin_constant_p(42)  // â†’ 0
```

### `__builtin_unreachable()`

Marks a point in the code that should never be reached. Compiled as a no-op (returns 0). In a production compiler this would enable dead code elimination.

### `__builtin_offsetof(type, member)`

Returns the byte offset of a struct member. Evaluated at compile time by looking up the member in the struct type's member list.

```c
struct S { int a; int b; };
__builtin_offsetof(struct S, b)  // â†’ 4
```

### `__builtin_types_compatible_p(type1, type2)`

Returns 1 if the two types are compatible (same kind and signedness), 0 otherwise. Evaluated at compile time.

```c
__builtin_types_compatible_p(int, int)   // â†’ 1
__builtin_types_compatible_p(int, long)  // â†’ 0
```

### `__builtin_va_list`

Recognized as a type name (equivalent to `va_list`), treated as `char*`.

### `__builtin_va_start`, `__builtin_va_arg`, `__builtin_va_end`, `__builtin_va_copy`

Aliases for the standard `va_start`, `va_arg`, `va_end`, and `va_copy` macros. Also handles `__va_copy`.

## Implementation

All built-ins are handled in `primary()` when an identifier matches a `__builtin_*` name:

- **`__builtin_expect`**: Parse two arguments, return the first
- **`__builtin_constant_p`**: Parse one argument, return `Num(0)`
- **`__builtin_unreachable`**: Parse empty parens, return `Num(0)`
- **`__builtin_offsetof`**: Parse type and member name, look up offset in struct members
- **`__builtin_types_compatible_p`**: Parse two types, compare their kind and signedness

## Test Cases

```c
int main() { return __builtin_expect(5, 1); }                              // â†’ 5
int main() { return __builtin_constant_p(42); }                            // â†’ 0
int main() { int x = 42; return __builtin_expect(x, 1); }                  // â†’ 42
struct S { int a; int b; }; int main() { return __builtin_offsetof(struct S, b); } // â†’ 4
struct S { int a; int b; }; int main() { return __builtin_offsetof(struct S, a); } // â†’ 0
int main() { return __builtin_types_compatible_p(int, int); }               // â†’ 1
int main() { return __builtin_types_compatible_p(int, long); }              // â†’ 0
```
