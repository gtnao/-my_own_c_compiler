# ステップ 14.16: `auto` ストレージクラス

## 概要

`auto` ストレージクラス指定子のサポートを追加する。読み取られた後、無視される。すべてのローカル変数はデフォルトで自動ストレージを持つ。

## なぜ必要か

`auto` が現代の C で明示的に使われることは稀だが、レガシーコードや一部の自動生成コードパターンに出現することがある。システムヘッダでも参照される場合がある。

## 実装

### トークン

`TokenKind` に `Auto` バリアントを追加した。

### 字句解析

`auto` をキーワードとして認識する。

### 構文解析

`auto` は `parse_type()` において `inline`、`register` およびその他のストレージ指定子と並んで読み取られ、無視される。`is_type_keyword()` および `stmt()` の型開始パターンにも追加した。

## 動作

- `auto int a = 5;` は `int a = 5;` と等価である
- `auto` キーワードは単に読み取られ、コード生成に影響を与えない

## テストケース

```c
int main() { auto int a = 5; return a; }  // → 5
```
