# Step 14.16: `auto` Storage Class

## Overview

Add support for the `auto` storage class specifier. It is consumed and ignored — all local variables have automatic storage by default.

## Why This Matters

While `auto` is rarely used explicitly in modern C, it can appear in legacy code and some generated code patterns. System headers may also reference it.

## Implementation

### Token

Added `Auto` variant to `TokenKind`.

### Lexer

Recognizes `auto` as a keyword.

### Parser

`auto` is consumed alongside `inline`, `register`, and other storage specifiers in `parse_type()`. Added to `is_type_keyword()` and `stmt()` type-start patterns.

## Behavior

- `auto int a = 5;` is equivalent to `int a = 5;`
- The `auto` keyword is simply consumed and has no effect on code generation

## Test Cases

```c
int main() { auto int a = 5; return a; }  // → 5
```
