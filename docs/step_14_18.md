# Step 14.18: Constant Expression Evaluation for Enum Values

## Overview

Enhance enum value parsing to support full constant expressions instead of just integer literals. This enables negative values, bitwise operations, arithmetic, and references to previously defined enum constants.

## Why This Matters

PostgreSQL enums use complex constant expressions:

```c
enum NodeTag {
    T_Invalid = 0,
    T_IndexInfo = 10,
    T_ExprContext,        // = 11 (auto-increment)
    T_ProjectionInfo,     // = 12
};

enum {
    PG_STRTOINT_OK = 0,
    PG_STRTOINT_OVERFLOW = -1,
    PG_STRTOINT_SYNTAX_ERROR = -2,
};

enum {
    FLAG_A = 1 << 0,     // = 1
    FLAG_B = 1 << 1,     // = 2
    FLAG_ALL = FLAG_A | FLAG_B,  // = 3
};
```

## Implementation

### `eval_const_expr()`

New method that parses a ternary expression and evaluates it at compile time:

```rust
fn eval_const_expr(&mut self) -> i64 {
    let expr = self.ternary();
    self.eval_const(&expr)
}
```

### `eval_const()`

Recursive constant expression evaluator that handles:

- **`Expr::Num(n)`** — integer literals (including hex/octal/binary)
- **`Expr::UnaryOp`** — negation (`-`), logical not (`!`), bitwise not (`~`)
- **`Expr::BinOp`** — all arithmetic (`+`, `-`, `*`, `/`, `%`), comparison (`==`, `!=`, `<`, `<=`, `>`, `>=`), and bitwise (`&`, `|`, `^`, `<<`, `>>`) operators
- **`Expr::LogicalAnd`/`LogicalOr`** — short-circuit logic
- **`Expr::Ternary`** — conditional expression
- **`Expr::SizeofType`** — compile-time sizeof
- **`Expr::Var(name)`** — lookup in enum_values (references to other enum constants)
- **`Expr::Cast`** — passes through to inner expression

### Enum Parsing Change

The enum value parsing now uses `eval_const_expr()` instead of a simple `Num(n)` match:

```rust
// Before:
if let TokenKind::Num(n) = self.current().kind {
    val = n;
    self.advance();
}

// After:
val = self.eval_const_expr();
```

This allows expressions like `1 << 3`, `-1`, `A + 3` (where A is a previous enum constant), etc.

## Test Cases

```c
enum { A = -1 }; int main() { return (unsigned char)A; }       // → 255
enum { A = 1 << 3 }; int main() { return A; }                  // → 8
enum { A = 2, B = A + 3 }; int main() { return B; }            // → 5
enum { A = 0xFF }; int main() { return A; }                     // → 255
enum { A, B, C }; int main() { return C; }                      // → 2
```
