# ステップ 14.18: enum 値のための定数式評価

## 概要

enum 値の解析を強化し、単なる整数リテラルではなく完全な定数式をサポートする。これにより、負の値、ビット演算、算術演算、および先に定義された enum 定数への参照が可能になる。

## なぜ必要か

PostgreSQL の enum では複雑な定数式が使われている:

```c
enum NodeTag {
    T_Invalid = 0,
    T_IndexInfo = 10,
    T_ExprContext,        // = 11 (auto-increment)
    T_ProjectionInfo,     // = 12
};

enum {
    PG_STRTOINT_OK = 0,
    PG_STRTOINT_OVERFLOW = -1,
    PG_STRTOINT_SYNTAX_ERROR = -2,
};

enum {
    FLAG_A = 1 << 0,     // = 1
    FLAG_B = 1 << 1,     // = 2
    FLAG_ALL = FLAG_A | FLAG_B,  // = 3
};
```

## 実装

### `eval_const_expr()`

三項演算式を解析してコンパイル時に評価する新しいメソッド:

```rust
fn eval_const_expr(&mut self) -> i64 {
    let expr = self.ternary();
    self.eval_const(&expr)
}
```

### `eval_const()`

以下を処理する再帰的な定数式評価器:

- **`Expr::Num(n)`** — 整数リテラル（16進数/8進数/2進数を含む）
- **`Expr::UnaryOp`** — 符号反転（`-`）、論理否定（`!`）、ビット反転（`~`）
- **`Expr::BinOp`** — すべての算術演算（`+`、`-`、`*`、`/`、`%`）、比較演算（`==`、`!=`、`<`、`<=`、`>`、`>=`）、ビット演算（`&`、`|`、`^`、`<<`、`>>`）
- **`Expr::LogicalAnd`/`LogicalOr`** — 短絡評価論理
- **`Expr::Ternary`** — 条件式
- **`Expr::SizeofType`** — コンパイル時の sizeof
- **`Expr::Var(name)`** — enum_values 内を検索（他の enum 定数への参照）
- **`Expr::Cast`** — 内部の式にパススルー

### enum 解析の変更

enum 値の解析で、単純な `Num(n)` マッチの代わりに `eval_const_expr()` を使用するようにした:

```rust
// Before:
if let TokenKind::Num(n) = self.current().kind {
    val = n;
    self.advance();
}

// After:
val = self.eval_const_expr();
```

これにより `1 << 3`、`-1`、`A + 3`（A が先に定義された enum 定数）などの式が使用可能になる。

## テストケース

```c
enum { A = -1 }; int main() { return (unsigned char)A; }       // → 255
enum { A = 1 << 3 }; int main() { return A; }                  // → 8
enum { A = 2, B = A + 3 }; int main() { return B; }            // → 5
enum { A = 0xFF }; int main() { return A; }                     // → 255
enum { A, B, C }; int main() { return C; }                      // → 2
```
