# ステップ 14.12: ビット演算複合代入演算子

## 概要

`&=`、`|=`、`^=`、`<<=`、`>>=` 複合代入演算子のサポートを追加する。これらは既存の算術複合代入（`+=`、`-=`、`*=`、`/=`、`%=`）と同様に `a = a op b` に展開される。

## なぜ必要か

PostgreSQL ではフラグ操作にビット演算複合代入が広く使われている:

```c
flags &= ~MASK;     // clear bits
flags |= FLAG;      // set bits
value <<= shift;    // shift left
hash >>= 1;         // shift right
```

## 実装

### トークン

5つの新しいトークン種別を追加した:
- `AmpEq` — `&=`
- `PipeEq` — `|=`
- `CaretEq` — `^=`
- `LShiftEq` — `<<=`
- `RShiftEq` — `>>=`

### 字句解析

5つの演算子すべての認識を追加した:
- `&=` と `|=` は2文字トークンで、`&&` と `||` よりも先にチェックされる
- `^=` は2文字トークンで、単独の `^` よりも先にチェックされる
- `<<=` と `>>=` は3文字トークンで、`<<` と `>>` よりも先にチェックされる

順序が重要である: `<<=` は `<<` よりも先にチェックする必要がある。そうしないとレキサーが `<<` を消費し、`=` が別のトークンとして残ってしまう。`>>=` と `>>` の関係も同様である。

### 構文解析

`assign()` の複合代入マッチに5つの新しい演算子を追加した:

```rust
let op = match self.current().kind {
    TokenKind::PlusEq => Some(BinOp::Add),
    TokenKind::MinusEq => Some(BinOp::Sub),
    TokenKind::StarEq => Some(BinOp::Mul),
    TokenKind::SlashEq => Some(BinOp::Div),
    TokenKind::PercentEq => Some(BinOp::Mod),
    TokenKind::AmpEq => Some(BinOp::BitAnd),
    TokenKind::PipeEq => Some(BinOp::BitOr),
    TokenKind::CaretEq => Some(BinOp::BitXor),
    TokenKind::LShiftEq => Some(BinOp::Shl),
    TokenKind::RShiftEq => Some(BinOp::Shr),
    _ => None,
};
```

各演算子は `a op= b` を `a = a op b` に展開し、既存の二項演算子のコード生成を再利用する。

## テストケース

```c
int main() { int a = 3; a &= 1; return a; }           // → 1
int main() { int a = 5; a |= 2; return a; }           // → 7
int main() { int a = 5; a ^= 3; return a; }           // → 6
int main() { int a = 1; a <<= 3; return a; }          // → 8
int main() { int a = 8; a >>= 2; return a; }          // → 2
int main() { int a = 0xFF; a &= 0x03; return a; }     // → 3
```
