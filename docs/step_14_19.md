# Step 14.19: Function Pointer Typedef and Array Typedef

## Overview

Add support for function pointer typedefs (`typedef void (*fn_ptr)(void)`) and array typedefs (`typedef int arr[10]`). Also fix `sizeof` to work with typedef names.

## Why This Matters

PostgreSQL uses function pointer typedefs extensively:

```c
typedef void (*pg_funcptr_t)(void);
typedef int (*comparator)(const void *, const void *);
typedef void (*MemoryContextCallbackFunction)(void *arg);
```

Array typedefs also appear:

```c
typedef int32 TransactionId;
typedef TransactionId MultiXactId;
```

## Implementation

### Function Pointer Typedef

When parsing a typedef, after getting the return type, check for the `(*Name)` pattern:

```rust
if self.current().kind == TokenKind::LParen
    && self.pos + 1 < self.tokens.len()
    && self.tokens[self.pos + 1].kind == TokenKind::Star
{
    // Parse: typedef RetType (*Name)(params...)
    self.advance(); // (
    self.advance(); // *
    // skip qualifiers
    let name = self.parse_ident();
    self.expect(TokenKind::RParen);
    // Skip parameter list
    self.skip_paren_group();
    self.expect(TokenKind::Semicolon);
    self.typedefs.insert(name, Type::ptr_to(return_type));
}
```

The function pointer is stored as `Ptr(return_type)` — a simplified representation that works for function pointer calls through the existing `FuncPtrCall` mechanism.

### Array Typedef

After getting the base type and name, check for `[N]`:

```rust
if self.current().kind == TokenKind::LBracket {
    self.advance();
    let size = self.eval_const_expr();
    self.expect(TokenKind::RBracket);
    let arr_ty = Type { kind: TypeKind::Array(Box::new(base), size), ... };
    self.typedefs.insert(name, arr_ty);
}
```

### sizeof Fix

Changed the `sizeof(type)` detection to use `is_type_start()` instead of `is_type_keyword()`, allowing sizeof to work with typedef names:

```rust
// Before: Self::is_type_keyword(&self.tokens[self.pos + 1].kind)
// After:  self.is_type_start(&self.tokens[self.pos + 1].kind)
```

This allows `sizeof(my_typedef)` to correctly look up the typedef and return the size.

### Both Top-Level and Statement-Level

Both the top-level `parse()` and statement-level `stmt()` typedef handlers now support function pointer and array patterns.

## Test Cases

```c
typedef void (*fn_ptr)(void); int main() { return 5; }    // → 5
typedef int (*binop)(int, int);
int add(int a, int b) { return a + b; }
int main() { binop f = add; return f(1, 2); }             // → 3
typedef int arr10[10]; int main() { return sizeof(arr10); } // → 40
```
