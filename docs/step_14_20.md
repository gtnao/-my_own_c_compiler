# ステップ 14.20: 構造体の前方宣言と void パラメータ

## 概要

構造体の前方宣言（`struct Foo;`）と、唯一の関数パラメータとしての `void`（「パラメータなし」を意味する `void func(void)`）のサポートを追加する。

## なぜ必要か

PostgreSQL では両方のパターンが広く使われている:

```c
// Forward declarations for mutually referential structs
struct ListCell;
struct List {
    int length;
    struct ListCell *head;
};
struct ListCell {
    int val;
    struct ListCell *next;
};

// void parameter meaning "no parameters"
void pg_init(void) { ... }
void MemoryContextReset(void) { ... }
```

## 実装

### 構造体の前方宣言

`parse_struct_or_union()` が、未知のタグの後に `;` が続く場合（またはボディ `{}` なしで型として使用される場合）、空の構造体プレースホルダーを作成する:

```rust
// Before: unknown tag → error
// After: unknown tag → create empty struct placeholder
None => {
    let ty = Type { kind: TypeKind::Struct(Vec::new()), is_unsigned: false };
    self.struct_tags.insert(tag.clone(), ty.clone());
    ty
}
```

後で完全な構造体定義が出現すると、プレースホルダーが完全な定義で上書きされる。これにより以下が可能になる:
- `struct Foo;` — 前方宣言
- `struct Foo *p;` — 前方宣言された構造体へのポインタ
- `struct Foo { int x; };` — 完全な定義がプレースホルダーを置き換える

### void パラメータ

C において `void func(void)` は「func はパラメータを取らない」ことを意味する。パーサーは次のようにチェックする: パラメータの型を解析した後、その型が `void` で次のトークンが `)` であれば、空のパラメータリストとして扱う:

```rust
let mut param_ty = self.parse_type();

// void as sole parameter means no parameters
if param_ty.kind == TypeKind::Void && self.current().kind == TokenKind::RParen {
    break;
}
```

これにより以下が正しく処理される:
- `void func(void)` → パラメータなし
- `void func(void *p)` → `void*` パラメータが1つ（`*` がポインタ型にするため、単なる void ではない）

### import の修正

`TypeKind::Void` 比較を可能にするために、パーサーの import に `TypeKind` を追加した:

```rust
use crate::types::{StructMember, Type, TypeKind};
```

## テストケース

```c
void noop(void) {} int main() { noop(); return 5; }                      // → 5
struct List; struct List { int val; }; int main() { struct List l; l.val = 3; return l.val; }  // → 3
```

## PostgreSQL パターン統合テスト

すべてのパターンが連携して動作することを検証する包括的なテスト:
- 構造体の前方宣言 + 自己参照ポインタ
- 負の値を持つ enum
- ビット演算を使った enum 値
- void パラメータ関数
- 関数ポインタ typedef
- 型エイリアス（typedef）
