# Step 14.20: Struct Forward Declarations and Void Parameter

## Overview

Add support for struct forward declarations (`struct Foo;`) and `void` as a sole function parameter meaning "no parameters" (`void func(void)`).

## Why This Matters

PostgreSQL uses both patterns extensively:

```c
// Forward declarations for mutually referential structs
struct ListCell;
struct List {
    int length;
    struct ListCell *head;
};
struct ListCell {
    int val;
    struct ListCell *next;
};

// void parameter meaning "no parameters"
void pg_init(void) { ... }
void MemoryContextReset(void) { ... }
```

## Implementation

### Struct Forward Declarations

When `parse_struct_or_union()` encounters an unknown tag followed by `;` (or used as a type without a body `{}`), it creates an empty struct placeholder:

```rust
// Before: unknown tag → error
// After: unknown tag → create empty struct placeholder
None => {
    let ty = Type { kind: TypeKind::Struct(Vec::new()), is_unsigned: false };
    self.struct_tags.insert(tag.clone(), ty.clone());
    ty
}
```

When the full struct definition appears later, it overwrites the placeholder with the complete definition. This allows:
- `struct Foo;` — forward declaration
- `struct Foo *p;` — pointer to forward-declared struct
- `struct Foo { int x; };` — full definition replaces placeholder

### Void Parameter

In C, `void func(void)` means "func takes no parameters". The parser now checks: after parsing a parameter type, if the type is `void` and the next token is `)`, treat it as an empty parameter list:

```rust
let mut param_ty = self.parse_type();

// void as sole parameter means no parameters
if param_ty.kind == TypeKind::Void && self.current().kind == TokenKind::RParen {
    break;
}
```

This correctly handles:
- `void func(void)` → no parameters
- `void func(void *p)` → one `void*` parameter (the `*` makes it a pointer type, not plain void)

### Import Fix

Added `TypeKind` to the parser's imports to enable the `TypeKind::Void` comparison:

```rust
use crate::types::{StructMember, Type, TypeKind};
```

## Test Cases

```c
void noop(void) {} int main() { noop(); return 5; }                      // → 5
struct List; struct List { int val; }; int main() { struct List l; l.val = 3; return l.val; }  // → 3
```
