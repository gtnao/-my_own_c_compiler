# Step 14.24: Function Pointer Calls Through Struct Members

## Overview

Add support for calling function pointers stored in struct members, both through direct member access (`s.fp(args)`) and through pointer member access (`p->fp(args)`). Also support function pointer member declarations in struct definitions.

## Why This Matters

PostgreSQL uses function pointer members in structs extensively:

```c
typedef struct FmgrInfo {
    PGFunction fn_addr;     // typedef for function pointer
    Oid fn_oid;
    short fn_nargs;
} FmgrInfo;

// Called as: info->fn_addr(fcinfo)

typedef struct MemoryContextMethods {
    void *(*alloc)(MemoryContext context, Size size);
    void (*free_p)(MemoryContext context, void *pointer);
    void (*reset)(MemoryContext context);
} MemoryContextMethods;

// Called as: context->methods->alloc(context, size)
```

## Implementation

### 1. Function Pointer Member in Struct Definition

When parsing struct members, after `parse_type()` returns the return type, check for the `(*name)(params)` pattern indicating a function pointer member:

```rust
if self.current().kind == TokenKind::LParen
    && self.pos + 1 < self.tokens.len()
    && self.tokens[self.pos + 1].kind == TokenKind::Star
{
    // Parse: return_type (*member_name)(param_types...)
    self.advance(); // (
    self.advance(); // *
    let name = self.parse_ident();
    self.expect(TokenKind::RParen);
    // Skip parameter list
    self.skip_paren_group();
    mem_ty = Type::ptr_to(Type::void()); // simplified representation
}
```

The function pointer is stored as `Ptr(Void)` — 8 bytes on x86-64, which is the correct size for any function pointer.

### 2. Function Pointer Call Through Member Access

In the `postfix()` parser, after constructing a `Member` expression from `.` or `->`, check if the next token is `(`:

```rust
TokenKind::Dot => {
    // ... parse member name, create Member node ...

    // Check for function pointer call: s.fp(args)
    if self.current().kind == TokenKind::LParen {
        self.advance();
        let args = self.parse_arg_list();
        self.expect(TokenKind::RParen);
        node = Expr::FuncPtrCall { fptr: Box::new(node), args };
    }
}
```

The same logic is applied to `->` (arrow) member access.

### How It Works

Given `s.fp(3, 4)`:

1. Parser creates `Member(Var("s"), "fp")` for the member access
2. Sees `(` after the member name → wraps in `FuncPtrCall`
3. Result: `FuncPtrCall { fptr: Member(Var("s"), "fp"), args: [3, 4] }`

Codegen for `FuncPtrCall`:
1. Evaluates `fptr` → loads the function pointer value from the struct member
2. Moves pointer to `%r10`
3. Sets up arguments in registers
4. Calls `*%r10`

## Test Cases

```c
// Function pointer member via typedef
unsigned long myfn(void *a) { return 42; }
typedef unsigned long (*fn_t)(void *);
typedef struct { fn_t fn; } S;
int main() { S s; s.fn = myfn; return (int)s.fn(0); }              // → 42

// Direct function pointer member syntax
int add(int a, int b) { return a+b; }
typedef struct { int (*f)(int, int); } S;
int main() { S s; s.f = add; return s.f(3, 4); }                   // → 7

// Arrow notation through pointer
int get(void) { return 10; }
typedef struct { int (*f)(void); } S;
int main() { S s; s.f = get; S *p = &s; return p->f(); }           // → 10
```
