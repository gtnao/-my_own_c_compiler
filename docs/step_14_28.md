# Step 14.28: Constant Expressions in Case Labels

## Overview

Allow case labels in switch statements to use constant expressions (enum constants, arithmetic, bitwise operations) instead of requiring plain integer literals.

## Why This Matters

PostgreSQL uses enum constants and computed values in switch statements extensively:

```c
switch (node->type) {
    case T_List:
    case T_IntList:
    case T_OidList:
        // ...
}

switch (flags) {
    case FLAG_A | FLAG_B:
        // ...
}
```

## The Bug

The case label parser only accepted `TokenKind::Num(n)`:

```rust
let val = match &self.current().kind {
    TokenKind::Num(n) => *n,
    _ => error("expected integer constant in case"),
};
```

This rejected enum constants (`case RED:`) and expressions (`case 1 << 3:`).

## Fix

Replace the simple number match with `eval_const_expr()`, which evaluates full constant expressions at compile time:

```rust
// Before:
let val = match &self.current().kind {
    TokenKind::Num(n) => *n,
    _ => error(...),
};
self.advance();

// After:
let val = self.eval_const_expr();
```

`eval_const_expr()` handles:
- Integer literals
- Enum constant references
- Arithmetic: `+`, `-`, `*`, `/`, `%`
- Bitwise: `&`, `|`, `^`, `~`, `<<`, `>>`
- Comparison: `==`, `!=`, `<`, `<=`, `>`, `>=`
- Logical: `&&`, `||`, `!`
- Ternary: `? :`
- Unary: `-`, `~`, `!`

## Test Cases

```c
enum { A, B, C }; int main() { int x = B; switch(x) {
    case A: return 0; case B: return 1; case C: return 2; } return -1; }  // → 1
int main() { int x = 3; switch(x) { case 1+2: return 30; default: return 0; } }  // → 30
int main() { int x = 8; switch(x) { case 1 << 3: return 8; default: return 0; } }  // → 8
```
