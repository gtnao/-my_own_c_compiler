# ステップ 14.28: caseラベルでの定数式

## 概要

switch文のcaseラベルで、単純な整数リテラルだけでなく定数式（列挙定数、算術演算、ビット演算など）を使用できるようにする。

## この実装が重要な理由

PostgreSQLではswitch文で列挙定数や計算された値が多用されている:

```c
switch (node->type) {
    case T_List:
    case T_IntList:
    case T_OidList:
        // ...
}

switch (flags) {
    case FLAG_A | FLAG_B:
        // ...
}
```

## バグの内容

caseラベルのパーサーは `TokenKind::Num(n)` のみを受け付けていた:

```rust
let val = match &self.current().kind {
    TokenKind::Num(n) => *n,
    _ => error("expected integer constant in case"),
};
```

そのため、列挙定数（`case RED:`）や式（`case 1 << 3:`）は拒否されていた。

## 修正内容

単純な数値マッチを、コンパイル時に完全な定数式を評価する `eval_const_expr()` に置き換えた:

```rust
// Before:
let val = match &self.current().kind {
    TokenKind::Num(n) => *n,
    _ => error(...),
};
self.advance();

// After:
let val = self.eval_const_expr();
```

`eval_const_expr()` は以下を処理する:
- 整数リテラル
- 列挙定数の参照
- 算術演算: `+`、`-`、`*`、`/`、`%`
- ビット演算: `&`、`|`、`^`、`~`、`<<`、`>>`
- 比較演算: `==`、`!=`、`<`、`<=`、`>`、`>=`
- 論理演算: `&&`、`||`、`!`
- 三項演算子: `? :`
- 単項演算子: `-`、`~`、`!`

## テストケース

```c
enum { A, B, C }; int main() { int x = B; switch(x) {
    case A: return 0; case B: return 1; case C: return 2; } return -1; }  // → 1
int main() { int x = 3; switch(x) { case 1+2: return 30; default: return 0; } }  // → 30
int main() { int x = 8; switch(x) { case 1 << 3: return 8; default: return 0; } }  // → 8
```
