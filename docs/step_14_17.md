# ステップ 14.17: `void *` ポインタ演算と宣言

## 概要

関数本体内での `void *` 変数宣言を修正し、`void *` ポインタ演算が正しく動作することを確認する（GCC 拡張: 演算目的で `sizeof(void)` を 1 バイトとして扱う）。

## なぜ必要か

PostgreSQL では汎用ポインタ操作に `void *` を広く使用している:

```c
void *ptr = palloc(size);
ptr = (char *)ptr + offset;  // or just ptr + offset with GCC extension
memcpy(dest, src, len);
```

## 実装

### 構文解析の修正

`stmt()` のマッチパターンに `TokenKind::Void` が欠落しており、関数本体内の `void *p = ...;` 宣言が "expected a number, identifier or '('" エラーで失敗していた。`stmt()` の型開始パターンに `TokenKind::Void` を追加した:

```rust
TokenKind::Void | TokenKind::Int | TokenKind::Char | ... => {
    self.var_decl()
}
```

### void ポインタ演算

`void *` ポインタ演算は、コード生成におけるポインタスケーリングの処理方法により、既に正しく動作していた:

```rust
if lhs_ty.is_pointer() {
    let size = lhs_ty.base_type().unwrap().size();
    if size > 1 {
        self.emit(&format!("  imul ${}, %rdi", size));
    }
}
```

`TypeKind::Void` は `size() == 0` であるため、条件 `size > 1` は偽となり、スケーリングは適用されない。これは `void *p + n` が `n` バイト分進むことを意味し、ポインタ演算において `sizeof(void) == 1` とする GCC の動作と一致する。

## 動作

- `void *p = &a;` — void ポインタの有効な宣言
- `p + 4` — 4バイト分進む（スケーリングなし、GCC 拡張の動作）
- `*(int *)p` — void ポインタを型付きポインタにキャストして参照外し

## テストケース

```c
int main() { int a[2]; a[0]=10; a[1]=20; void *p=&a[0]; p=p+4; return *(int*)p; }  // → 20
int main() { int a = 10; void *p = &a; return *(int*)p; }                            // → 10
```
